# 阅读顺序

这是会话接班的强制阅读顺序。

## 第 0 步：启动
1. `tools/qf init`

## 第 1 步：同频层（必须按顺序）
1. `SYNC/00_同频入口.md`
2. `SYNC/02_当前状态.md`
3. `SYNC/03_本次会话.md`
4. `SYNC/04_最新决策.md`
5. `SYNC/05_索引链接.md`

## 第 2 步：治理核心
1. `AGENTS.md`
2. `docs/WORKFLOW.md`
3. `docs/ENTITIES.md`
4. `docs/PROJECT_GUIDE.md`
5. `TASKS/STATE.md`
6. `TASKS/QUEUE.md`

## 第 3 步：当前 RUN 证据
1. 最新 `reports/<RUN_ID>/handoff.md`（如果存在）
2. 最新 `reports/<RUN_ID>/conversation.md`（如果存在）
3. 最新 `reports/<RUN_ID>/decision.md`
4. 最新 `reports/<RUN_ID>/summary.md`

## 完成标准（必须复述）
- 目标（一句话）
- Scope（精确路径）
- 验收（verify/evidence/scope）
- 下一条命令（只给一条）
