## 决策 1：learn 必须支持真实模型同频
- 结论：`tools/qf learn` 引入 Codex real interaction（read-only + JSONL + output-last-message）并写入模型证据文件。
- 原因：同频不能只靠本地脚本拼接，必须有真实模型口述与问答结果。
- 影响：learn 可审计性显著提升，且支持强模式门禁。

## 决策 2：learn 强模式必须输出口述与问答锚点
- 结论：强模式要求 plan_protocol + oral_restate + oral_exam（>=3 QA）。
- 原因：防止“同频完成但不可解释”。
- 影响：控制台可直接看到主线、阶段、下一步与口述问答。

## 决策 3：同频考试升级为 15+2 深度问卷
- 结论：用你定义的问题体系替换旧考试模板与 rubric。
- 原因：旧模板对“项目理解/主线承接”覆盖不足。
- 影响：考试通过标准更贴近真实上岗能力。

## 决策 4：旧答卷自动迁移
- 结论：`exam-auto` 在自动填充模式下，检测到旧格式答卷会自动重填。
- 原因：减少版本切换带来的人工阻塞。
- 影响：learn 自动链路稳定性提高。

## 决策 5：新增 Codex CLI 操作手册
- 结论：新增 `docs/CODEX_CLI_OPERATION.md`，统一讨论/执行模式与参数语义。

- 影响：Codex 使用方法可复用、可培训、可审计。
